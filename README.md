# py_art

Small 2019 experiments in generating procedural, geometric "art" backgrounds
with text stamped on top, using Pillow. Originally written to produce the
featured images for blog posts on [m7c1.com](https://m7c1.com) — the tiled,
mirrored, Material-Design-colored images behind post titles were all
generated by `py_art_dev.py`.

Ported to Python 3 (from the original Python 2 / Pillow ~5.x code) with
minimal changes — mostly `print` statements, integer division, and
replacing a couple of Pillow APIs (`ImageDraw.textsize`, `Image.ANTIALIAS`)
that were removed in modern Pillow. No behavior/logic changes beyond
normalizing a few `draw.rectangle` calls that could otherwise pass inverted
coordinates and crash under newer, stricter Pillow versions.

## Scripts

- **`py_art_dev.py`** — the finished generator. Tiles a grid of colored
  squares (drawn from two randomly chosen Material Design palettes) with
  small decorative pieslices/rectangles, optionally mirrors the left half
  onto the right, auto-sizes the title text to fit, and stamps it on top
  with a black stroke outline. This is what actually produced the m7c1
  post images.
- `3_try.py`, `2_try.py`, `stylized_text1.py` — earlier iterations kept for
  history, each a step toward `py_art_dev.py`'s approach. Not configurable
  from the command line — edit the `text`/`width`/`height` variables at
  the top of the file directly.

## Requirements

```
pip install -r requirements.txt
```

A DejaVu Sans Bold TTF is expected at
`/usr/share/fonts/truetype/dejavu/DejaVuSans-Bold.ttf` (installed by
default on most Debian/Ubuntu systems via `fonts-dejavu-core`). Edit the
`dejavu_sans_bold` path in the script if yours lives elsewhere.

## Usage

```
./py_art_dev.py "<text>" <mirror> <square_scale>
```

- `text` — the title to render.
- `mirror` — `1` to mirror the left half of the generated background onto
  the right half, `0` to draw the full width independently.
- `square_scale` — side length (px) of the tiling squares. Smaller values
  give a busier, more detailed background; larger values give bigger,
  blockier tiles.

Output is saved as `<text_with_underscores>_image.png` in the current
directory.

### Examples

```
./py_art_dev.py "Friedrich Hayek" 1 100
./py_art_dev.py "Buffer Overflow" 0 60
./py_art_dev.py "Tor" 1 150
```

Run it a few times with the same text — the palette pair and tile
placement are randomized on every run, so no two outputs are identical.
